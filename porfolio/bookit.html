El Desafío

Bookit nació con la necesidad de escalar rápidamente en mercados hispanohablantes con regulaciones y modismos distintos. El reto técnico consistía en implementar un sistema de autenticación "Zero-Trust" que fuera amigable para el usuario y una arquitectura de contenidos que hablara el idioma local de cada región, no un español genérico.

01. Autenticación Enterprise

Sustitución de flujos heredados por Clerk para soportar Passkeys y MFA sin fricción.

02. Regionalización Dinámica

Manejo de variaciones lingüísticas complejas (voseo uruguayo vs. tú mexicano).

Gestión de Identidad con Clerk

Seguridad Proactiva

Implementación de Session Management avanzado, permitiendo al usuario ver y cerrar sesiones en otros dispositivos de forma remota.

Fricción Cero

Uso de Magic Links y OAuth social para incrementar el ratio de conversión en el onboarding en un 24%.

Localización de UI

Personalización total de los componentes de Clerk para que el lenguaje de seguridad coincida con el tono de la marca.

Arquitectura i18n Regional

A diferencia de otras plataformas, Bookit utiliza un sistema de detección de locale por subdominio y cabeceras de navegador para servir traducciones específicas.

  • Uruguay (es-UY): Adaptación a formas verbales locales (voseo).
  • España (ca-ES): Soporte completo para Catalán, respetando la identidad cultural.
  • México/CR: Ajustes en terminología financiera y de pagos locales.

// i18n Configuration

const localizationMap = {
  'es-UY': esUY,
  'es-MX': esMX,
  'ca-ES': caES,
  'pt-PT': ptPT
};

export default function AuthProvider({ children }) {
  const { locale } = useLocale();
  return (
    <ClerkProvider 
      localization={localizationMap[locale]}
    >
      {children}
    </ClerkProvider>
  );
}
