Auditoría en ambiente de Producción

La auditoría separa falsos positivos de navegador, warnings de terceros y problemas reales del sitio para priorizar acciones de bajo riesgo y alto impacto operativo.

Criticidad baja

Permissions Policy

Los mensajes web-share y compute-pressure provienen del entorno de YouTube o de permisos no habilitados para el documento.

Criticidad media

CSP mal configurada

La consola repite que frame-ancestors contiene report-to como si fuera un origen. Esto indica una cabecera Content-Security-Policy construida con sintaxis incorrecta.

Criticidad media

Origen YouTube inconsistente

El warning de postMessage muestra que la API de YouTube intenta comunicarse con un origen que no coincide con el iframe cargado.

Criticidad media

Performance

El log repite advertencias de recursos cargados con link preload que no se usan pocos segundos despues del load. Esto puede indicar preloads excesivos, atributo as incorrecto o assets priorizados sin necesidad.

Criticidad alta

SEO / GEO

El sitio necesita reforzar metadatos, schemas y señales de entidad para mejorar lectura orgánica y consistencia en buscadores.

Criticidad alta

Dependencias expuestas

La extension VULNERS detecta Font Awesome, Yoast, jQuery, jQuery Migrate y WordPress. No todo es falla real, pero exige validar versiones.

Criticidad alta

Formulario Error 404

El formulario no está disponible en la sección contacto. Falta endpoint, campos, validacion y prueba de envio.

Criticidad alta

WidgetApi Downtime

El widget de WhatsApp no está 24/7 activo, forzando a los usuarios a utilizar otros canales de comunicación, y reduciendo la eficiencia en la atención al cliente.

Criticidad alta

Subcontratación de servicios

La subcontratación de servicios externos puede generar riesgos de seguridad y calidad en la entrega de soluciones digitales. Además, suma riesgo de dependencia tecnológica.

Planes

Cada plan tiene un objetivo de negocio claro: estabilizar la operación digital o estabilizarla y mejorar la percepción de velocidad en mobile con métricas comparables.

Plan 1

Plan Integral

Base mínima viable para estabilizar WordPress, reducir alertas visibles, activar canales de contacto y dejar el sitio trazable para mantenimiento.

Alcance técnico

  • Auditoria WordPress, plugins, tema, cache y dependencias visibles.
  • Revision de consola: VULNERS, YouTube, CSP, Permissions Policy y preloads.
  • Correccion o documentacion de warnings accionables.
  • Integracion Formspree para formulario de contacto.
  • SmartWeb Widget WhatsApp.
  • SmartWeb SEO Schema para metas y JSON-LD complementario.
  • Alta y monitoreo inicial con Modular DS.

Entregables y continuidad

  • Informe tecnico con hallazgos y acciones aplicadas.
  • Formulario y WhatsApp listos para captura de consultas.
  • Base SEO estructurada y documentada.
  • Tablero Modular DS conectado.
  • Recomendaciones para sostenibilidad anual.

Plan 2

Plan Integral + Core Web

Plan recomendado cuando se quiere convertir la auditoría en una mejora perceptible para usuarios móviles, con foco en PageSpeed, LCP, Speed Index y recursos críticos.

Alcance técnico

  • Todo lo incluido en el Plan Integral.
  • Auditoria comparativa PageSpeed mobile/desktop.
  • Refactor de CSS no utilizado detectado por PageSpeed.
  • Optimizacion de LCP, Speed Index mobile y recursos above the fold.
  • Revision de fuentes, preload/preconnect y lazy loading.
  • Correcciones menores de accesibilidad y SEO tecnico.

Entregables y continuidad

  • Todo el paquete documental del Plan Integral.
  • Comparativa mobile/desktop antes y despues.
  • Detalle de assets intervenidos.
  • Reporte de mejoras Core Web aplicadas.
  • Lista de optimizaciones futuras si exceden el alcance.
